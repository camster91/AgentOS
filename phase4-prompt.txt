## Step 1: Weather Agent

Create core/src/main/kotlin/com/agentOS/agents/WeatherAgent.kt

Requirements:
- Extends Agent class
- Uses Open-Meteo API (free, no API key needed): https://api.open-meteo.com/v1/forecast
- Uses OpenStreetMap Nominatim for geocoding (city name → lat/lon): https://nominatim.openstreetmap.org/search
- Constructor takes StorageAPI (use NoOpAgentAPI pattern from CalendarAgent)
- AgentScope: id="com.agentOS.weather", name="Weather Agent", capabilities=setOf("storage", "ui")
- HTTP calls: use java.net.HttpURLConnection (no extra dependencies)
- Cache city coordinates in storage (avoid hammering Nominatim)
- Nominatim API: GET https://nominatim.openstreetmap.org/search?q=<city>&format=json&limit=1
  - Returns JSON array, parse first result for "lat" and "lon" fields
  - IMPORTANT: Set User-Agent header to "AgentOS/0.1.0" (Nominatim requires this)
- Open-Meteo API: GET https://api.open-meteo.com/v1/forecast?latitude=<lat>&longitude=<lon>&current_weather=true&daily=temperature_2m_max,temperature_2m_min,weathercode&timezone=auto&forecast_days=7
  - Parse current_weather object for temperature and windspeed
  - Parse daily arrays for forecast

Parse JSON manually using string operations (no external JSON library):
- For Nominatim: extract "lat" and "lon" from first JSON object in array
- For Open-Meteo: extract from current_weather object

onChat() commands:
- "weather <city>" → current weather for city
- "forecast <city>" → 3-day forecast
- "weather tomorrow" → tomorrow's weather (use last queried city or ask for city)
- anything else → help text listing commands

Store last queried city in storage so "weather tomorrow" works without specifying city.

Format output nicely:
- "Current weather in Toronto: 15°C, Wind 20 km/h"
- "3-day forecast for Toronto:\n  Today: 12-18°C\n  Tomorrow: 10-16°C\n  Day 3: 9-14°C"

## Step 2: Tasks Agent  

Create core/src/main/kotlin/com/agentOS/agents/TasksAgent.kt

Requirements:
- Extends Agent class
- Constructor takes StorageAPI
- AgentScope: id="com.agentOS.tasks", name="Tasks Agent", capabilities=setOf("storage", "ui")
- Task data: id (UUID-style string), title, priority (LOW/MEDIUM/HIGH), dueDate (epoch ms, optional), completed (boolean), createdAt (epoch ms)
- Store tasks as simple delimited strings in StorageAPI (same pattern as NotesAgent)
- Parse due dates: "today", "tomorrow", "next Monday", "Jan 15", ISO date format (YYYY-MM-DD)
  - Use java.time.LocalDate for date calculations
- Priority parsing: "high", "medium", "low" in the command

onChat() commands:
- "add task: <title> [priority: high/medium/low] [due: <date>]" → creates task
- "add task <title>" → creates task with MEDIUM priority and no due date
- "list tasks" → shows all incomplete tasks with IDs, priorities, due dates
- "complete task <id or title>" → marks task as complete
- "due today" or "tasks due today" → shows tasks due today
- "due this week" → shows tasks due in next 7 days
- "delete task <id or title>" → removes task
- "all tasks" → shows all tasks including completed
- anything else → help text

## Important notes:
- WeatherAgent HTTP calls: use java.net.URL.openConnection() as HttpURLConnection
- Set timeouts: connectTimeout = 5000, readTimeout = 10000
- Handle network errors gracefully (return error message string, don't crash)
- TasksAgent date parsing: use java.time.LocalDate (available in JVM 8+)
- No external JSON or HTTP libraries — pure JVM standard library only
- All code stays in :core module (no new submodules)
- Do not modify Agent.kt, AgentAPI.kt, AgentScope.kt in :api module
- The NoOpAgentAPI pattern is already in core/src/main/kotlin/com/agentOS/agents/NoOpAgentAPI.kt — use it
